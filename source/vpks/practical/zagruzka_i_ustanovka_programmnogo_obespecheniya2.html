Лабораторная работа. Виртуальные машины

Цель работы: Рассмотреть работу виртуальной машины. Научиться подключать виртуальные машины и диски. Ознакомиться с интерфейсом различных систем, запускаемых в виртуальной машине.

 ПОДГОТОВКА К РАБОТЕ

Виртуальная машина (ВМ, от англ. virtual machine) – программная и/или аппаратная система, эмулирующая аппаратное обеспечение некоторой (target, целевой, гостевой) платформы и исполняющая программы для этой платформы на (базовой, платформе-хозяине)host-платформе.ВМ работает как отдельная программа и позволяет запускать одну операционную систему в другой в виде отдельного приложения параллельно другим исполняемым программам базовой системы. Программы гостевой системы изолированы внутри виртуальной машины.

В качестве ВМ выбрана бесплатная программа Oracle VirtualBox54. Данная ВМ позволяет запускать множество операционных систем, в том числе различные версии Windows и Linux, подготовленные дистрибутивы которых используются в данной лабораторной работе.

Подготовленные виртуальные машины хранятся в локальной папке D:\Документы\VM, а на сервере домена в подключённом сетевом ресурсе «X:\Учебные материалы\VM\» хранится эталон, используемый в случае повреждения локальной копии. Папкой для размещения может быть любая папка на томе локальной машины с достаточным объёмом свободного пространства. Для запуска виртуальной машины необходимо распаковать файл диска с ВМ, зарегистрировать, и в случае необходимости настроить ВМ. После проведения лабораторной работы необходимо отменить регистрацию и удалить рабочие файлы из папки на локальном диске (архив не удалять!). Для создания новой виртуальной машины вне лабораторной работы необходимо иметь дистрибутив требуемой системы, создать и установить систему на виртуальный диск.

Периодически производителем виртуальной машины версии программы обновляются55, поэтому при выполнении работы некоторые позиции или наименования настройки могут не точно совпадать с приведенными в этом руководстве, поэтому при отсутствии указанной позиции требуется внимательно рассмотреть вкладки управления машиной для нахождения искомой настройки.

Ввиртуальную машину можно установить пакет расширений (VirtualBox Extension Pack), который добавляет дополнительные возможности: удалённое управление виртуальными машинами, виртуальный USB 2.0 порт, загрузочное ПЗУ сетевой карты, сквозное использование мыши и клавиатуры в базовой и гостевой системе (иначе мышь и клавиатура захватываются гостевой системой). Пакет расширения загружается с сайта виртуальной машины и должен иметь ту же версию, что и виртуальная машина. Пакет расширения устанавливается в виде дополнительных драйверов в гостевые системы.

Вкачестве управляющей кнопки, позволяющей управлять и выйти из виртуальной машины, назначена правая кнопка Windows.

При создании новой ВМ и распаковке готового файла виртуального диска следите за количеством свободного места на рабочем разделе, т.к. объём файлов виртуальных дисков достаточно велик. В случае нехватки места удалите ВМ в списке VirtualBox с указанием удалять и файлы.

ЗАПУСК LINUX В КАЧЕСТВЕ ГОСТЕВОЙ СИСТЕМЫ

Распакуйте файл виртуального раздела Kubintu.7z в папке D:\Документы\VM (в случае отсутствия или повреждения скопируйте в неё файл архива из эталона, см. выше). Запустите программу VirtualBox. Для создания/регистрации виртуальной машины в основном окне VirtualBox выберите команду Машина/Создать (Ctrl+N). В качествеимени машины56 введите Linux Kubuntu, в полеТип ОС выберите

Операционная система – Linux,версия – Ubuntu (буква перед названием сборки Ubuntu уточняет конфигурацию сборки, в частности K обозначает

использование среды рабочего стола KDE вместо Unity. KDE по внешнему виду ближе к интерфейсу Windows, что снижает затраты времени на обучение персонала при переходе на эту систему). В следующем окне укажите (цифры можно ввести с клавиатуры) требуемый объём оперативной памяти – 512Мб. Далее, в следующем окне, установите галочку Загрузочный диск, выберитеИспользовать существующий жёсткий диск, нажмите кнопку справа списка дисков

D:\Документы\VM\Kubuntu.vdi. Подтвердите окончание создания виртуальной машины кнопкой Create (Создать/Готово). Выберите в списке созданную виртуальную машину «Linux Kubuntu» и выполните командуМашина/Свойства (Ctrl+S). Проверьте настройку следующих свойств виртуальной машины (остальные параметры остаются с настройками по умолчанию):

Раздел Система– вкладка Материнская плата, список Порядок загрузки– оставьте галочку выбора загрузки только у жёсткого диска. В области Дополнительные возможности выключите параметр

Абсолютные устройства позиционирования.

Раздел Носители – в блокеНосители информации нажмите правой кнопкой мыши на строкуSATA контроллер, выберите командуудалить контроллер. Нажмите правой кнопкой мыши на строкуIDE

контроллер, выберите команду Добавить жёсткий диск, Выбрать существующий диск, выберите D:\Документы\VM\Ubuntu\ Kubuntu.vdi. При выделении в списке устройства Kubuntu.vdi в атрибутах жёсткого диска должен указываться Первичный мастер IDE, при выделении строки компакт-диска – Вторичный мастер IDE.

Раздел Общие папки – с помощью кнопки справаДобавить общую папку добавьте папку D:\Документы, дайтеимя папки (ресурса) DOC,

установите Только для чтения (что предотвратит изменение документов изнутри виртуальной машины). Расположение курсора

Запустите виртуальную машину.

После загрузки системы с помощью меню команд виртуальной машины раскройте окно виртуальной машины на весь экран. Найдите, запустите и рассмотрите интерфейс работы калькулятора, файлового менеджера (раздел системного меню Приложения/Стандартные), также рассмотрите в разделе Приложения/Офис интерфейс электронных таблиц LibreOffice Calc и текстового процессора LibreOffice Writer.

Выключите гостевую систему Linux Kubuntu командойСистема/Выключить. Программу VirtualBox в базовой системе выгружать не нужно.

Распакуйте архив Linux.7z, содержащий файл виртуального диска с предустановленной системой Linux Mandriva. Самостоятельно создайте виртуальную машину, указав в качестве диска файл «Linux Mandriva.vdi» и объем оперативной памяти 384Мб. Запустите Linux Mandriva (имя/пароль: stud/stud), сравните интерфейсы версий. Выключите гостевую систему. Повторите эту операцию для файла Ubuntu.7z.
